# Правила документирования кода — Doc Master

## Python: Google-style Docstrings

Каждая функция, метод, класс и модуль ОБЯЗАНЫ иметь docstring в формате Google.

### Шаблон для функций/методов:
```python
def calculate_ccc(receivables_days: int, inventory_days: int, payables_days: int) -> float:
    """Рассчитывает цикл конвертации денежных средств (CCC).

    Цикл конвертации денежных средств показывает, сколько дней компания
    преобразует инвестиции в запасы и другие ресурсы в денежные потоки.

    Args:
        receivables_days: Дни дебиторской задолженности (DSO).
        inventory_days: Дни в запасах (DIO).
        payables_days: Дни кредиторской задолженности (DPO).

    Returns:
        Значение CCC в днях (float). Отрицательное — позитивный знак.

    Raises:
        ValueError: Если переданы отрицательные значения дней.

    Example:
        >>> calculate_ccc(45, 30, 60)
        15.0
    """
```

### Шаблон для классов:
```python
class SalesRepository:
    """Репозиторий для работы с данными продаж из базы данных.

    Реализует паттерн Repository для изоляции логики доступа к данным.
    Все методы асинхронные для совместимости с FastAPI async runtime.

    Attributes:
        db: Асинхронная сессия SQLAlchemy.
    """
```

### Обязательные требования Python:
- **Type hints**: ВСЕ параметры и возвращаемые значения ОБЯЗАНЫ иметь аннотации типов.
- **Optional**: Использовать `Optional[T]` или `T | None` для опциональных параметров.
- **Return types**: Никогда не оставлять без `-> ReturnType`.
- **Pydantic models**: Поля с `Field(description="...")` на русском языке.

```python
# ❌ Неправильно:
def get_product(id):
    pass

# ✅ Правильно:
async def get_product(product_id: uuid.UUID) -> ProductSchema | None:
    """Возвращает продукт по идентификатору."""
```

## React: TSDoc для компонентов

Каждый React-компонент ОБЯЗАН иметь TSDoc-комментарий.

### Шаблон для компонента:
```tsx
/**
 * Карточка KPI с анимированным счётчиком и иконкой тренда.
 *
 * @component
 * @example
 * ```tsx
 * <KpiCard
 *   title="Выручка"
 *   value={1250000}
 *   trend={+12.5}
 *   currency="KZT"
 * />
 * ```
 */
interface KpiCardProps {
  /** Заголовок метрики (например, "Выручка за период") */
  title: string;
  /** Числовое значение метрики */
  value: number;
  /** Изменение в процентах относительно предыдущего периода */
  trend?: number;
  /** Код валюты (KZT, USD, EUR) */
  currency?: string;
}
```

### Обязательные требования React/TypeScript:
- **Props interface**: Каждый компонент имеет именованный `interface ComponentNameProps`.
- **Default exports**: Компоненты экспортируются как `export default`.
- **Named exports**: Утилиты и хуки экспортируются как именованные.
- **JSDoc для хуков**: Хуки документируются аналогично функциям Python.
- **No `any`**: Запрещено использование типа `any`. Использовать `unknown` с guard'ами.

## Общие правила

1. **Язык документации**: Русский — для всей документации, комментариев, описаний.
2. **Язык кода**: Английский — для имён переменных, функций, классов, файлов.
3. **TODO-комментарии**: Формат `# TODO(автор): описание задачи` с указанием задачи.
4. **Устаревший код**: Использовать `@deprecated` с пояснением альтернативы.
5. **Бизнес-логика**: Сложные формулы (EBITDA, CCC, GMROI) ОБЯЗАТЕЛЬНО документируются с математическими формулами в комментариях.
